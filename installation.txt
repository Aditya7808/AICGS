CareerBuddy Installation Guide (Supabase Version)
=========================================================

PREREQUISITES:
- Python 3.8 or higher
- Node.js 16 or higher  
- Supabase account (free at supabase.com)
- Git

QUICK START (Automated Migration):
1. Run the migration script:
   ./migrate_to_supabase.sh

2. Follow the prompts to configure your .env file

3. Access the application:
   - Frontend: http://localhost:5173
   - Backend: http://localhost:8000/docs

MANUAL INSTALLATION:

1. CLONE REPOSITORY:
   git clone <repository-url>
   cd careerbuddy

2. SUPABASE SETUP:
   - Create account at https://supabase.com
   - Create new project
   - Get credentials from Settings > API
   - Copy database URL from Settings > Database

3. CONFIGURE ENVIRONMENT:
   cp backend/.env.template backend/.env
   # Edit .env with your Supabase credentials

4. BACKEND SETUP:
   cd backend
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   pip install -r requirements.txt
   python setup_supabase_complete.py

5. FRONTEND SETUP:
   cd frontend
   npm install

6. START APPLICATION:
   # Terminal 1 (Backend):
   cd backend
   source venv/bin/activate
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

   # Terminal 2 (Frontend):
   cd frontend
   npm run dev

7. ACCESS APPLICATION:
   - Frontend: http://localhost:5173
   - Backend API: http://localhost:8000
   - API Documentation: http://localhost:8000/docs

MIGRATION FROM SQLITE:
If you had the previous SQLite version:
1. Your old SQLite files will be automatically cleaned up
2. Data can be migrated using the migration script in SUPABASE_MIGRATION_COMPLETE.md
3. All new features now use Supabase PostgreSQL

ENVIRONMENT VARIABLES (.env file):
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key  
DATABASE_URL=postgresql://postgres:password@db.your-project-ref.supabase.co:5432/postgres
SECRET_KEY=your-secure-secret-key-minimum-32-characters

TROUBLESHOOTING:
- Database connection errors: Check DATABASE_URL format
- Authentication errors: Verify Supabase keys
- Import errors: Ensure all requirements are installed
- Port conflicts: Change ports in startup commands
